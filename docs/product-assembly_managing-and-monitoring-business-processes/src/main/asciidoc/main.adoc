// Purpose statement for the assembly
As a process administrator who needs to easily monitor the state of all running processes across a number of projects, you can use {CENTRAL} in {PRODUCT} manage and monitor process instances and tasks. From {CENTRAL} you can start a new process instance, verify the state of all process instances, and abort processes. You can view the list of jobs and tasks associated with your processes, as well as understand and communicate any process errors. 

.Prerequisites
* Installed {EAP_LONG} 7.1.0. See  https://access.redhat.com/documentation/en-us/red_hat_jboss_enterprise_application_platform/7.1/html-single/installation_guide/[_Red Hat JBoss EAP 7.1.0 Installation Guide_].
* Installed {PRODUCT}. For more information, see the {URL_INSTALLING_DM_ON_PREMISE}[{INSTALLING_DM_ON_PREMISE}].
* {PRODUCT} is running and you can log in to {CENTRAL} with the `process-admin` role. For more information, see the {URL_INSTALLING_DM_ON_PREMISE}[{INSTALLING_DM_ON_PREMISE}].
